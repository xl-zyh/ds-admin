## 1. 系统概述

### 1.1 业务背景

全民健身信息服务平台是国家级体育场馆客流数据汇聚平台。各场馆通过本地客流系统采集数据后，按规范要求向上级平台推送数据，实现全国场馆客流数据的统一管理与分析。

### 1.2 数据流向

```
场馆本地系统 → 聚合/转换 → HTTP POST → 全民健身平台 → 数据校验 → 入库
```

## 2. 技术规范

### 2.1 基础环境

| 项目 | 规范 |
|------|------|
| 测试环境 | `http://open.js365.org.cn/api/customerflow/` |
| 传输协议 | HTTP/1.1 over TLS（生产环境建议 HTTPS） |
| 请求方式 | POST（获取令牌为 GET） |
| 字符编码 | UTF-8 |
| 数据格式 | JSON（Body 部分）；坐标接口为 form-data |
| 认证方式 | Bearer Token（Header: `Authorization`） |

### 2.2 编码标准

- **行政区划编码**：参照 `GB/T 2260-2007`（中华人民共和国行政区划代码）
- **场馆编号**：参照《体育场馆信息化管理服务系统技术规范》
- **用户识别方式编码**：参照规范 `6.1.5` 场馆用户识别编码
- **性别编码**：参照规范 `6.1.6` 性别编码

### 2.3 时间格式规范

| 场景 | 格式 | 示例 |
|------|------|------|
| 小时段数据采集时间 | `yyyy-MM-dd HH:mm:ss` | `2026-06-16 18:00:00`（代表 17:00-18:00） |
| 5分钟粒度采集时间 | `yyyy-MM-dd HH:mm:ss` | `2026-06-16 18:05:00`（代表 18:00-18:05） |
| 每日汇总日期 | `yyyy-MM-dd HH:mm:ss` | `2026-06-16 23:50:00` |
| 用户出生日期 | `yyyy-MM-dd` | `1990-01-15` |

### 2.4 字段约束约定

| 符号 | 含义 |
|------|------|
| `M` | 必填（Mandatory） |
| `0` | 非必填（Optional） |

## 3. 认证与安全

### 3.1 接入申请流程

```
场馆单位 → 向管理部门申请数据接入授权 → 审核通过 → 发放 appId + appSecret
```

### 3.2 获取令牌（Token）

**请求方式**：`GET`

**请求地址**：
```
http://open.js365.org.cn/api/token?grant_type=client_credential&Appid={appId}&Secret={appSecret}
```

| 参数 | 必填 | 说明 |
|------|------|------|
| `grant_type` | 是 | 固定值 `client_credential` |
| `Appid` | 是 | 接入时分配的 appId |
| `Secret` | 是 | 接入时分配的 appSecret |

**成功响应**：
```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "expires_in": 86400
}
```

| 字段 | 说明 |
|------|------|
| `access_token` | 令牌凭证，后续所有接口调用时携带 |
| `expires_in` | 有效期，单位：秒（86400 = 24小时） |

### 3.3 令牌错误码

| 返回码 | 说明 | 处理建议 |
|--------|------|----------|
| `-1` | 系统繁忙 | 稍后重试，建议间隔 30s |
| `0` | 请求成功 | — |
| `40001` | AppSecret 错误或不属于该场馆 | 检查 appSecret 是否正确 |
| `40002` | grant_type 不是 client_credential | 检查参数拼写 |
| `40003` | 无效 appId | 检查 appId 是否正确 |
| `40004` | 调用 IP 不在白名单中 | 联系管理员配置 IP 白名单 |

### 3.4 请求鉴权

所有数据上报接口均需在 HTTP Header 中携带令牌：

```
Authorization: Bearer {access_token}
Content-Type: application/json
```

### 3.5 安全建议

1. **令牌缓存**：令牌有效期 24h，建议缓存并在过期前 5 分钟刷新
2. **IP 白名单**：申请时将服务器出口 IP 加入白名单
3. **重试策略**：令牌过期后重新获取，不要使用旧令牌重试
4. **敏感信息**：appSecret 不要硬编码在代码中，通过环境变量或配置中心管理

## 4. 数据模型设计

### 4.1 通用字段说明

以下字段在所有接口中含义一致：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `provinceId` | Int | 省份编号，GB/T 2260-2007 |
| `cityId` | Int | 城市编号，GB/T 2260-2007 |
| `districtId` | Int | 区县编号，GB/T 2260-2007 |
| `venueId` | String | 场馆编号，参照体育场馆技术规范 |
| `isCoreArea` | Int | 是否核心免低区域：1=是，0=否 |

## 5. 接口详细定义

> **Base URL**：`http://open.js365.org.cn/api/customerflow`
>
> 所有接口请求 Header：
> ```
> Authorization: Bearer {access_token}
> Content-Type: application/json
> ```

### 5.1 二维码签到系统数据接口（8.4.1）

```
POST /syncQRCodeFlowData
```

**说明**：数据以小时为单位统计，`acqTime` = 18:00 表示 17:00-18:00 时段。

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `provinceId` | Int | M | 省份编号 |
| `cityId` | Int | M | 城市编号 |
| `districtId` | Int | M | 区县编号 |
| `venueId` | String | M | 场馆编号 |
| `acqTime` | String | M | 采集时间 `yyyy-MM-dd HH:mm:ss` |
| `acqNumOfPeople` | Int | M | 入场人数 |
| `leaNumOfPeople` | Int | M | 出场人数 |
| `deviceId` | String | M | 设备编号 |
| `isCoreArea` | Int | M | 是否核心免低区域：1=是，0=否 |

**请求示例**：
```json
{
  "provinceId": 330000,
  "cityId": 330100,
  "districtId": 330102,
  "venueId": "VEN20260001",
  "acqTime": "2026-06-16 18:00:00",
  "acqNumOfPeople": 120,
  "leaNumOfPeople": 80,
  "deviceId": "QR-DEV-001",
  "isCoreArea": 1
}
```

### 5.14 图片上传 OBS 路径获取（附录）

```
POST /syncImgObsUrl

```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `venueId` | String | M | 场馆编号 |

**响应**：`msg` 字段返回 OBS 图片上传路径。

**图片上传流程**：
1. 调用 `/syncImgObsUrl` 获取 OBS 上传 URL
2. 使用 HTTP PUT 请求将图片上传到 OBS URL
3. 上传成功后，URL 去掉 `?AccessKeyId=...` 参数即为最终图片路径
4. 将该路径填入 8.4.8 接口的 `imgUrl` 字段

**OBS 上传代码示例（Java）**：
```java
// 1. 获取上传 URL
String obsUrl = callSyncImgObsUrl(venueId); // msg 字段的值

// 2. PUT 上传图片
File file = new File("capture.png");
String contentType = "image/png";
Request request = new Request.Builder()
    .url(obsUrl)
    .put(RequestBody.create(MediaType.parse(contentType), file))
    .build();
OkHttpClient client = new OkHttpClient();
Response response = client.newCall(request).execute();

// 3. 获取最终图片路径
if (response.code() == 200) {
    String imgUrl = response.request().url().toString();
    imgUrl = imgUrl.substring(0, imgUrl.indexOf("?AccessKeyId"));
    // 使用 imgUrl 填入 8.4.8 接口
}
```

### 6.3 数据聚合与转换

**小时粒度聚合伪代码**（适用于 8.4.1 ~ 8.4.5）：

```python
from collections import defaultdict
from datetime import datetime, timedelta

def aggregate_hourly(raw_records, target_hour_end):
    """
    将原始明细记录聚合为小时粒度
    raw_records: [(timestamp, in_count, out_count, device_id, ...), ...]
    target_hour_end: 时段结束时间，如 18:00
    """
    hour_start = target_hour_end - timedelta(hours=1)
    filtered = [r for r in raw_records if hour_start <= r.timestamp < target_hour_end]

    return {
        "acqTime": target_hour_end.strftime("%Y-%m-%d %H:%M:%S"),
        "acqNumOfPeople": sum(r.in_count for r in filtered),
        "leaNumOfPeople": sum(r.out_count for r in filtered),
        "deviceId": filtered[0].device_id,  # 聚合维度按设备
        "isCoreArea": 1 if filtered[0].is_core else 0,
    }
```

### 6.4 推送容错与重试

```
┌─────────────────────────────────────────────────────┐
│                    推送流程                           │
├─────────────────────────────────────────────────────┤
│  1. 从本地 DB 读取待推送数据                          │
│  2. 检查 token 是否有效（过期前 5 分钟刷新）           │
│  3. 发起 HTTP POST                                   │
│  4. 检查响应                                          │
│     ├── 200 OK → 标记推送成功，记录推送时间            │
│     ├── 4xx → 检查 token → 刷新后重试（最多 1 次）    │
│     ├── 5xx → 指数退避重试（最多 3 次，间隔 5s/15s/30s）│
│     └── 网络异常 → 同上重试 + 本地持久化待推送队列     │
│  5. 重试 3 次后仍失败 → 记录异常日志 + 告警通知        │
└─────────────────────────────────────────────────────┘
```

## 7. 错误处理机制

### 7.1 平台返回错误码

| 返回码 | 含义 | 处理方式 |
|--------|------|----------|
| `0` | 请求成功 | 正常处理 |
| `-1` | 系统繁忙 | 30s 后重试 |
| `40001` | AppSecret 错误 | 检查配置，不重试 |
| `40002` | grant_type 错误 | 检查参数，不重试 |
| `40003` | 无效 appId | 检查配置，不重试 |
| `40004` | IP 不在白名单 | 联系管理员添加 IP |

### 7.2 客户端错误处理

| 场景 | 策略 |
|------|------|
| 网络超时 | 连接超时 10s，读取超时 30s；超时后重试 |
| DNS 解析失败 | 重试 3 次，间隔 5s |
| Token 过期（401） | 重新获取 token 后重试 1 次 |
| 请求频率过高（429） | 等待 Retry-After 秒后重试 |
| 服务端错误（500） | 指数退避重试，最多 3 次 |
| 参数校验失败（400） | 记录错误日志，不重试，人工排查 |

### 7.3 日志规范
